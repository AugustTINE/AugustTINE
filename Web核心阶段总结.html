Web核心阶段总结
  • Web核心阶段三大组件：Servlet/JSP、Filter、Listener
  • Servlet/JSP
    • 基本概念：Servlet本质是运行在服务端的java程序，用户通过URL地址，以HTTP协议的方式远程调用Servlet的API，Servlet可以向客户端动态响应数据
    • 作用：接受数据；处理请求；响应数据
    • Servlet接口的方法：5个，重要的有3个
      • init(ServletConfig config)，初始化方法
        • config包含Servlet对象的配置信息，由服务器创建并传递过来；一般在Servlet对象中定义一个成员变量用于接收ServletConfig
        • 通过ServletConfig对象可以获得以下数据：
          • Servlet名称
          • Servlet的初始化参数
          • 获取ServletContext对象
      • service(ServletRequest req,ServletResponse res)，接收请求、响应数据的方法，参数由服务器传递得到
        • 主要方法
          • String req.getParameter(String name),获取名称为name的请求参数，参数值只有一个
          • String[] req.getParameterValues(String name),获取名称为name的请求参数，参数值为多个
          • PrintWriter res.getWriter(),获取响应流，有writer()、print()方法可以用来向客户端响应数据
      • destroy方法，Servlet正常销毁时执行destroy方法
      • Servlet的生命周期
        • 在首次被访问时，Servlet对象被创建，执行init方法进行初始化，init方法仅在创建Servlet对象时执行一次
        • 以后的每次访问，都是执行service方法，但每次的request和response对象都是不一样的
        • 项目被移除或者服务器被关闭时，Servlet对象被销毁
    • Servlet配置xml
      • <servlet>
        • <servlet-name></servlet-name>
        • <servlet-class>Servlet全路径名</servlet-class>服务器根据类的路径，使用反射的方式在内存中创建Servlet对象
      • <servlet-mapping>：配置Servlet对象的映射路径
        • <servlet-name></servlet-name>
        • <url-pattern></url-pattern>：配置Servlet的访问入口，配置方式有4种
          • 完全匹配：/servletDemo1，在浏览器地址栏输入：/项目名/servletDemo1可访问该Servlet
          • 目录匹配，以/开头，以*结尾：/aa/bb/*、/download/*，表明所有访问地址是项目名/aa/bb/任意字符 的路径都能访问到该Servlet
          • 拓展名匹配，以*开头，以拓展名结尾：*.jsp、*.abc，表明访问地址的拓展名是.jsp的路径都能访问到该Servlet
          • 缺省匹配/
          • 注意点：目录匹配和拓展名匹配不能混用；路径优先级为完全匹配>目录匹配>拓展名匹配>缺省匹配；尽量不使用缺省匹配
    • HttpServlet：实现Servlet接口的方式使用起来不方便，实际应用中通过继承HttpServlet类代替实现Servlet接口的方式
      • 继承体系
      • ServletContext对象，在服务器启动时为web应用创建，封装了该应用的整体的、宏观的信息，在应用任意地方可以获取到该对象
        • 获取ServletContext对象：ServletContext getServletContext();
        • ServletContext中的方法
          • 获取全局的初始化参数
            • String getInitParameter(String name);获取名称为name的初始化参数的值
            • Enumeration<String> getInitParameterValues(String name);
              • 获取名称为name的初始化参数的枚举对象
              • 通过hasMoreElements()、nextElement()方法对枚举对象进行遍历和取值
          • 获取项目中任意资源的实际路径
            • String context = this.getServletContext();获取ServletContext对象
            • String path = context.getRealPath("资源的服务端路径");获取的路径是资源的实际路径，即Tomcat本地目录下的资源地址
            • InputStream is = context.getResourceAsStream("资源的服务端路径");读取资源到输入流中，可以用来向客户端响应某类文件资源
          • 属性的方法
            • 每个域对象都包括3个属性的操作方法
            • void setAttribute(String name,Object obj);设置属性
            • Object getAttribute(String name);根据属性名获取属性
            • void removeAttribute(String name);根据属性名移除属性
      • ServletResponse对象
        • 设置响应行的操作：设置响应状态码
          • setStatus(int code);仅响应状态码，即使发送的是错误代码，浏览器也不会报错
          • sendError(int code);如果响应错误代码，浏览器会报错
        • 设置响应头的操作
          • setHeader(String name,String value);
            • setHeader("set-cookie","name=value;name=value...");
            • setHeader("Content-Disposition","attachment;filename="+文件名);指定客户端处理响应体的方式为附件，返回响应文件的文件名
          • addHeader(String name,String value);追加响应头
          • sendRedirct(String path);重定向302，path为客户端地址，客户端接到该响应后会再次向path发起一次新的请求
        • 设置响应体
          • 字符型响应体
            • PrintWriter getWriter();通过打印流写出的数据会存储在response缓冲区中
            • response缓冲区的编码集是iso-8859-1，不支持中文，需要在获取打印流之前设置响应内容的类型和浏览器的解码方式:setContentType("text/html;charset=utf-8");
          • 字节型响应体
            • ServletOutputStream getOutputStream();可以用来响应文件、图片
        • 文件下载案例
      • ServletRequest对象
        • 获取请求行的操作
          • 获取请求方式：String getMethod();请求方式有get、post等
          • 获取请求的资源路径
            • String getRequestURI();
            • String getRequestURL();
          • 获取请求的应用名称：String getContextPath();ServletContext域对象也有获取应用名称的方法，但是需要首先获取ServletContext对象，实际中主要使用request对象的方法获取
        • 获取请求头：String getHeader(String name);
        • 获取请求参数
          • String getParameter(String name);获取名称为name的请求参数的值
          • String[] getParameterValues(String name);获取名称为name的请求参数的数组
          • Map<String,String[]> getParameterMap();获取请求参数的Map对象，value值是String数组
        • 属性的三个方法，setAttribute(String name,Object obj);getAttribute(String name);removeAttribute(String name);
        • 请求转发：
          • request.getRequestDispatcher("服务端路径名").foward(request,response);
          • 可以将数据存储在request域对象中，通过请求转发在Web应用内部传递数据
      • Cookie/Session对象：会话管理
        • cookie的创建与删除
          • void setMaxAge(int seconds);设置Cookie对象的持久化时间，将同名、同路径的Cookie对象的有效期设置为0并响应给浏览器，可以达到删除cookie的目的
          • void setPath("客户端路径");默认是创建该cookie的资源所在的路径，可以通过setPath方法手动指定可访问该cookie的路径
          • response.addCookie(new Cookie("name","value"));将Cookie对象返回给浏览器，下次请求时浏览器会自动携带cookie数据给服务器
        • cookie的获取方法
          • Cookie[] cookies = request.getCookies();获取请求中的所有cookie数组，通过遍历数组可以获取每个cookie对象及其name属性和value属性
          • for(Cookie cookie : cookies){String cookieName = cookie.getName(); String cookieValue = cookie.getValue();}
        • Session对象
          • 服务端在首次执行request.getSession()方法时，会创建session对象，同时在该次响应中向浏览器发送包含SESSIONID的set-cookie响应头，将SESSIONID存储在客户端
          • session是域对象，可以通过操作属性的方法存储数据：session.setAttribute(String name,Object value);也可以或者指定name的属性值
          • session默认有效时间为30m，可以在xml中配置有效时间，单位是分钟；也可以在通过void setMaxInactiveInterval(int seconds);方法设置有效时间，单位是秒，getMaxInactiveInterval();方法可以获得session有效时间的参数（秒）
    • JSP
      • 内置对象（9个）
        • 内置对象是用来方便脚本语言操作的预定义对象，在JSP被Tomcat引擎转换为Servlet时，Servlet的service方法会创建这9个内置对象
          • 4个域对象
            • application，相当于ServletContext对象
            • request；session
            • pageContext，当前页面的对象，可以通过pageContext对象获取Servlet中域对象
          • out，用来输出的对象；response
          • config；page；exception，page属性设置了isErrorPage="true"，才能使用exception对象
      • 指令：<%@ 指令名称 属性1="属性值1" 属性2="属性值2" ...%>
        • page：<%@ page %>，告诉JSP引擎如何将JSP文件转换为Servlet，属性有import（导包），language，charset等
        • include：<%@ include file="被包含页面路径"%>，用来包含其他页面，包括JSP页面、HTML页面，底层原理是在转换JSP页面时将被包含页面的代码复制进来，形成一个Servlet文件；因此，在包含页面与被包含页面中，不能重复定义某个同名的变量
        • taglib：<%@ taglib%>，用来引入其他标签库，比如JSTL的Core标签库<%@ taglib uri="http://java.sun.com/jsp/jstl/core" prefix="c"%>
      • 脚本元素
        • <%! java代码 %>：用来定义成员方法或者变量，JSP转换为Servlet时会被写入Servlet的成员区域，可以定义成员方法、成员变量、内部类
        • <% java代码 %>：用来写普通的java语句，JSP转换为Servlet时会作为java代码被写入service方法中，可以定义局部变量、语句
        • <%=java表达式 %>：用来输出结果，必须是能计算出结果的表达式；JSP转换为Servlet时表达式的结果会作为输出内容写入service方法的out.print("计算结果")语句中
      • 动作：
        • <jsp:forward page="服务端跳转路径"></jsp:forward>，实现请求转发
        • <jsp:include page="被引用页面路径">，include动作的原理是在响应时调用其他页面的结果，生成各自的java文件，独立存在
    • EL/JSTL：
      • EL：表达式语言
        • 主要作用有：1.快速获取应用程序数据；2.进行表达式运算
        • 内置对象：11个
          • pageContext：JSP的页面域对象，可以用来获取JSP的9个内置对象（pageContext/request/session/application/out/response/config/page/exception）
          • initparam：环境的初始化参数Map，用参数名作为key
          • param/paramValues：请求参数的Map，参数名作为key，paramValues的Map的V是String数组，可以用来获取参数名相同的所有参数值
          • header/headerValues：请求头的Map，标题名作为key，headerValues的Map的V也是String数组，可以用来获取指定标题名的所有参数值
          • pageScope/requestScope/sessionScope/applicationScope，域对象的Map，包含了域对象的所有属性；表示的范围不同
        • 取值方式：
          • ${域对象的Map.属性名}：${requestScope.username}，在指定的范围内查找属性
          • ${属性名}，按照pageContext-->request-->session-->application顺序知道找到该属性，都没有找到则返回null,${cookie.SESSIONID.value}
        • EL运算符
          • 算术运算符：+、-、*、/和div、%和mod
          • 逻辑运算符：&&和and、||和or、！和not
          • empty运算符：对象为null或者字符串为空时，${empty x}为true
      • JSTL：JSP标准标签库：
        • if标签：<c:if test="表达式">表达式为true时执行的代码</c:if>
        • forEach标签
          • 格式：<c:forEach items="数组/List/Set/Map" [var="varName"] [begin="begin"] [end="end"] [step="step"] [varStatus="varStatusName"]>body</c:forEach>
          • 可以用遍历数组、集合、列表，Map，也可以通过设置begin和end属性实现指定范围的遍历，类似于java代码中的for(int i=0;i<10;i++){body};
          • varStatus表示被取出的元素的状态信息，varStatusName.count 表示遍历到了第几次；varStatusName.first/last 表示当前遍历元素是否是第一个/最后一个被遍历元素
  • Filter过滤器
    • 概念：过滤器用来拦截、筛选用户请求，并作出相应放行或者其他处理
    • 过滤器链：多个过滤器的拦截范围一样，xml中配置的顺序不一样，会按照先后顺序执行，形成过滤器链；过滤器链之间形成"可插拔式"的功能组件，可以方便地进行添加和移除组件的操作
    • 执行流程
      • 加载过滤器：服务器启动时，根据xml文件的配置，将字节码文件加载到内存，通过反射创建各个过滤器对象，每个过滤器对象中的init初始化方法被执行一次
      • 当客户端请求到达服务器时，如果配置的过滤器路径与请求的路径匹配，执行过滤器中的doFilter(ServletRequest request, ServletResponse response, FilterChain chain)方法
      • 如果执行到chain.doFilter(request,response)方法，相当于放行，请求的目标资源被执行（目标资源指存储在服务的并请求的资源，可以是HTML、Servlet、JSP等）
      • 目标资源执行完成之后，返回过滤器，继续执行chain.doFilter(request,response)方法后面的代码
      • 如果在执行过滤器中doFilter方法时没有遇到放行语句，Tomcat仅仅对本次请求响应 HTTP响应部分的响应行、响应头信息
    • xml配置
      • <filter>
        • <filter-name></filter-name>
        • <filter-class>filter包名.类名</filter-class>指定过滤器，用于在内存中通过反射创建对象
      • <filter-mapping>：过滤器映射入口
        • <filter-name></filter-name>
        • <url-pattern>拦截的路径，匹配方式有：1.目录匹配：/*、/download/*；2.拓展名匹配：*.jsp、*.avi；3.完全路径匹配(较少使用)：/aaa </url-pattern>
  • Listener
    • 基本概念
      • 事件源：被监听的对象
      • 监听器：监听事件源的某类事件的发生
      • 事件：被监听事件发生时执行的操作
    • JavaWeb监听器的种类
      • 第一类：监听域对象的创建和销毁：Lifecycle
        • 监听ServletContext的创建和销毁，对应实现ServletContextListener接口
        • 监听session的创建和销毁，对应实现HttpSessionListener接口
        • 监听request的创建和销毁，对应实现ServletRequestListener接口
      • 第二类：监听域对象的属性变动：Change to attributes（大Map）
        • 监听ServletContext域对象的属性变动
        • 监听session域对象的属性变动
        • 监听request域对象的属性变动
      • 第三类：监听JavaBean对象在session中的状态变更（绑定到某个session，解除绑定；钝化到磁盘，活化到内存）
        • Object binding监听JavaBean对象绑定与解绑
          • 监听JavaBean对象绑定值某个session：request.getSession().setAttribute("user",new User());
          • 监听JavaBean对象从某个session移除：request.getSession().removeAttribute("user");
        • Session migration监听JavaBean对象钝化与活化
          • 钝化：服务器正常关闭，session数据会被写到硬盘中，存储在session中的JavaBean对象也会随着session写入硬盘（JavaBean对象需要实现序列化接口）
          • 活化：服务器再次启动，从对象位置读取session数据，JavaBean对象随着session被再次读取到内存中
      • 三类监听器总结
        • 监听器会在服务器启动时创建，生命周期与Web项目一致
        • 前两类监听器是创建Listener，Eclipse自动实现对应的接口；而第三类监听器是在JavaBean类上实现对应接口：HttpSessionBindingListener
        • 前两类监听器需要自动或手动地配置xml，而JavaBean对象的监听不需要配置xml
        • 应用场景
          • ServletContext域对象的监听：ServletContext对象随着项目启动创建，可以用来监听项目的创建，执行某些需要项目一启动就执行的代码
          • Session域对象的监听：
            • 监听Session对象的创建和销毁可以用来估计服务器当前访问量
            • 在某些时候监听session对象的销毁，执行持久化session中数据的操作，比如购物车的数据
          • Request域对象的监听：可以通过监听request的创建，估计瞬时访问量，估计系统压力
          • 第三类监听可以了解JavaBean对象在session中的使用情况
  • 其他
    • JavaWeb的三大组件都是单实例；
    • Servlet是单实例多线程，多个线程同时访问的是同一个Servlet对象
